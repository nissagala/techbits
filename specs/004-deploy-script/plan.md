## Summary

Create a single bash script at `/home/nissanka/.local/bin/update-tech-live` that syncs the TechBits Laravel application from the local development machine (`/home/nissanka/MSC/techbits`) to the production server (`techlive@47.131.59.99:/home/nginx/sites/techbits_live`) via `rsync`, then runs all required remote post-deploy steps (composer, migrations, optional seeding, cache rebuild, permissions) over a single SSH session. The script uses `set -euo pipefail` + an ERR trap for safe error handling and maintenance mode management.

## Script Architecture

### Configuration block (top of script)

```bash
# ─── Configuration ────────────────────────────────
REMOTE_USER="techlive"
REMOTE_HOST="47.131.59.99"
REMOTE_PATH="/home/nginx/sites/techbits_live"
LOCAL_PATH="/home/nissanka/MSC/techbits"
RUN_SEEDS=false       # set to true to also run db:seed --force
```

### rsync exclusions

The following paths are **never overwritten** on the remote server:

```
.env
storage/
vendor/
.git/
node_modules/
```

### Deployment sequence

```
[local]  1. rsync local → remote (with exclusions, progress output)
[remote] 2. php artisan down          ← maintenance mode ON
[remote] 3. composer install --no-dev --optimize-autoloader
[remote] 4. php artisan migrate --force
[remote] 5. php artisan db:seed --force   ← only if RUN_SEEDS=true
[remote] 6. php artisan config:cache
[remote] 7. php artisan route:cache
[remote] 8. php artisan view:cache
[remote] 9. php artisan event:cache
[remote] 10. chmod -R ug+rwx storage/ bootstrap/cache/
[remote] 11. php artisan up           ← maintenance mode OFF
```

**Why rsync before maintenance mode**: Syncing PHP files while the app is live is safe (files are read atomically per-request). Maintenance mode activates before composer/migrate — the operations that can leave the app in a broken mid-state.

### Error handling

```bash
set -euo pipefail

CURRENT_STEP=""

on_error() {
    echo ""
    echo "[✗] Deployment FAILED at step: ${CURRENT_STEP}"
    echo "    Attempting to restore app from maintenance mode..."
    ssh "${REMOTE_USER}@${REMOTE_HOST}" \
        "cd '${REMOTE_PATH}' && php artisan up 2>/dev/null || true"
    echo "    Done. Check output above for details."
    exit 1
}

trap on_error ERR
```

### Progress output format

```
[→] Syncing files to remote server...
[✓] Files synced
[→] Enabling maintenance mode...
[✓] Maintenance mode ON
[→] Installing dependencies...
[✓] Dependencies installed
...
[✓] Deployment complete. App is live.
```
